## 方法二：本地运行项目
本项目使用 HBuilderX 打开。

打开步骤：
下载HBuilderX
→ 文件
→ 打开目录
→ 选择本项目文件夹

点击运行
→ 运行到浏览器
→ Edge 或 Chrome
注意：不要直接双击 index.html 打开，否则页面可能无法正常跳转。

## 前端和后端怎么联调

前端接口统一写在：
```text
utils/api.js
```

当前本地接口地址：

```js
const BASE_URL = 'http://127.0.0.1:8000'
```

如果后端同学在本地运行后端，前端可以连接本地接口。

如果要让 GitHub Pages 在线预览也能访问真实接口，后端需要部署公网地址，例如：

```js
const BASE_URL = 'https://你的后端公网地址'
```

修改后，需要重新打包 H5，并更新 `docs` 文件夹。


## 说明

1. 不要直接修改 `docs` 里的页面源码  `docs` 是打包后的网页文件，真正的源码在 `pages`、`utils`、`static` 等文件夹里。

2. 修改源码后，在线预览不会自动更新。需要重新打包发行，并把发行后的文件覆盖 `docs` 文件夹。

3. `unpackage` 不需要上传，它是本地打包生成的临时目录。

4. 如果接口失败，不一定是前端问题。可能是后端服务没有启动，或者后端地址不是公网地址。

5. 如果页面样式显示异常，请先用 HBuilderX 重新运行到浏览器，不要直接打开本地 `index.html`。
